# Quickstart

This guide walks you through submitting a render job and retrieving the output URL.

## Prerequisites

- A FrameLane API key (get one from the [Console](https://console.framelane.io))
- Python 3.9+ or Node.js 18+

## 1. Install the SDK

<CodeGroup>

```bash Python
pip install framelane
```

```bash TypeScript
npm install framelane
```

</CodeGroup>

## 2. Create a render job

Reference your source media by URL — FrameLane fetches it directly, no upload step required.

<CodeGroup>

```python Python
from framelane import FrameLane

client = FrameLane()  # reads FRAMELANE_API_KEY from env

job = client.jobs.create(
    assets=["https://cdn.framelane.io/raw-footage.mp4"],
    edits=[
        {"type": "trim", "start": 0, "end": 30},
        {"type": "caption", "style": "subtitles"},
    ],
    output={"format": "mp4", "resolution": "1080p"},
)
print(f"Job ID: {job.id}")
```

```typescript TypeScript
import { FrameLane } from "framelane";

const client = new FrameLane(); // reads FRAMELANE_API_KEY from env

const job = await client.jobs.create({
  assets: ["https://cdn.framelane.io/raw-footage.mp4"],
  edits: [
    { type: "trim", start: 0, end: 30 },
    { type: "caption", style: "subtitles" },
  ],
  output: { format: "mp4", resolution: "1080p" },
});
console.log(`Job ID: ${job.id}`);
```

</CodeGroup>

<Note>
  Passing a URL renders that file once. If you plan to reuse the same media
  across multiple jobs, [upload it as an asset](/concepts/assets) first and
  reference the returned `asset_id` instead — it's stored once and reusable.
</Note>

## 3. Wait for completion

<CodeGroup>

```python Python
result = client.jobs.wait(job.id)
print(f"Output: {result.output_url}")
```

```typescript TypeScript
const result = await client.jobs.wait(job.id);
console.log(`Output: ${result.outputUrl}`);
```

</CodeGroup>

<Tip>
  For production use, configure [webhooks](/concepts/webhooks) instead of
  polling so your agent reacts to events in real-time.
</Tip>
