# Assets

An **Asset** is any media file uploaded to FrameLane — raw footage, audio tracks, logos, fonts, or graphics packs. Assets are the inputs to render jobs.

## Creating assets

Upload a file to receive an Asset object with a unique `id`, metadata (duration, resolution, codec), and a storage URL.

<CodeGroup>

```python Python
asset = client.assets.upload("raw-footage.mp4")
```

```typescript TypeScript
const asset = await client.assets.upload("raw-footage.mp4");
```

</CodeGroup>

FrameLane auto-detects format, duration, resolution, and codec on upload. Supported containers include MP4, MOV, WebM, MKV, and more.

## Asset metadata

Every asset includes:

| Field        | Description                          |
|--------------|--------------------------------------|
| `id`         | Unique identifier (`ast_...`)        |
| `filename`   | Original upload filename             |
| `mime_type`  | Detected MIME type                   |
| `duration`   | Duration in seconds (video/audio)    |
| `resolution` | Width x height (video/image)         |
| `codec`      | Detected codec                       |
| `size_bytes` | File size                            |
| `created_at` | Upload timestamp                     |

## Lifecycle

Assets are retained for the duration of your plan's storage quota. Delete assets explicitly when no longer needed to free storage.
